# Мониторинг ресурсов в Linux

Заметка про то, как читать `top`/`htop` и понимать общую нагрузку на систему.

---

## Load average

Вверху `top`/`htop` строка `load average: 0.25, 0.40, 0.60` показывает среднюю нагрузку за **1, 5 и 15 минут**.

- Числа показывают, **сколько задач одновременно хотят выполняться** (стоят в очереди к CPU или I/O).  
- На однопроцессорной системе `load = 1.0` ≈ все ресурсы заняты; на 4‑ядерной машине нормой считается до ~4.0.  
- Если load стабильно намного больше количества ядер — есть постоянная очередь задач, система «тупит».

Пример на 4‑ядерном сервере:
- `load 1.0` — ядра простаивают  
- `load 4.0` — все ядра загружены  
- `load 8.0` — половина задач ждёт очереди

## us, sy, wa, id — расшифровка CPU

Строка `Cpu(s): 5.6%us, 3.0%sy, 90.9%id, 0.5%wa`:

| Поле | Что показывает | Что значит высокий процент |
|------|----------------|----------------------------|
| **us** | пользовательский код | много вычислений в приложениях |
| **sy** | код ядра | много системных вызовов |
| **id** | простой CPU | ресурсов хватает |
| **wa** | ожидание I/O | тормозит диск/сеть |

**Выводы:**
- Высокий `wa` + высокий load → проблемы с диском/сетью  
- Высокий `us`/`sy` → CPU‑нагрузка  
- Высокий `id` → можно грузить больше

---

## Что ещё смотреть в top/htop

- **RAM/Swap** — активный swap = не хватает памяти  
- **RES/RSS** у процессов — реальное потребление памяти  
- Сортировка по CPU/памяти для поиска «прожорливых» процессов

# Быстрый конспект по Git для начинающих

Коротко о том, что нужно знать в начале, чтобы уверенно работать с Git.

## Что такое Git
- Распределённая система контроля версий: хранит историю изменений файлов, позволяет работать локально и синхронизироваться с удалёнными репозиториями.
- Репозиторий = папка с проектом + скрытая папка `.git`, где Git хранит историю.

## Установка и первая настройка (Windows)
- Скачайте и установите Git: https://git-scm.com/downloads
- Настройка пользователя (в терминале):
  - git config --global user.name "Ваше Имя"
  - git config --global user.email "you@example.com"
- Проверьте версию:
  - git --version

## Основные понятия
- Commit — снимок состояния (фиксирование изменений).
- Branch — ветка разработки.
- Remote — удалённый репозиторий (например, GitHub).
- Staging area (индекс) — подготовка изменений к коммиту.
- Merge / Rebase — способы объединения изменений.
- .gitignore — список файлов/папок, которые не нужно отслеживать.

## Базовые команды (рабочий процесс)
- Инициализация репозитория:
  - git init
- Клонирование удалённого репозитория:
  - git clone <url>
- Проверка состояния:
  - git status
- Добавление изменений в индекс:
  - git add <файл> или git add .
- Коммит:
  - git commit -m "Короткое сообщение"
- Просмотр истории:
  - git log --oneline --graph --decorate
- Показать изменения:
  - git diff (неиндексированные), git diff --staged (индексированные)

## Работа с ветками
- Создать и перейти в ветку:
  - git checkout -b feature
- Переключиться:
  - git checkout main
- Объединить изменения (в ветке, куда вливаем):
  - git merge feature
- Удалить ветку:
  - git branch -d feature

## Удалённые репозитории (push / pull)
- Добавить remote:
  - git remote add origin <url>
- Отправить ветку:
  - git push -u origin main
- Получить изменения и слить:
  - git pull (fetch + merge)
- Получить без слияния:
  - git fetch
  - git merge origin/main или git rebase origin/main

## Отмена изменений и восстановление
- Отменить изменения в рабочем каталоге (неиндексированные):
  - git checkout -- <файл>
- Убрать из индекса (оставив файл в рабочем каталоге):
  - git reset HEAD <файл>
- Откатить последний коммит, сохранив изменения в файлах:
  - git reset --soft HEAD~1
- Откатить и удалить изменения окончательно:
  - git reset --hard HEAD~1 (Осторожно!)

## Stash — временно сохранить изменения
- Сохранить текущие изменения:
  - git stash
- Применить последний stash:
  - git stash apply
- Просмотреть список:
  - git stash list

## Теги (tags)
- Создать тег:
  - git tag v1.0
- Отправить теги на remote:
  - git push origin --tags

## .gitignore
- Создайте файл `.gitignore` в корне и перечислите шаблоны файлов/папок, которые не надо отслеживать (например, `node_modules/`, `*.log`, `dist/`).

## Советы по коммитам и веткам
- Делайте небольшие, осмысленные коммиты.
- Сообщения: короткая первая строка (~50 символов), при необходимости — тело с деталями.
- Используйте ветки для фич, багфиксов и экспериментов.
- Перед merge делайте git pull и решайте конфликты локально.

## Git в Visual Studio Code
- Панель Source Control (Ctrl+Shift+G) для основных операций.
- Встроенный терминал: Ctrl+` 
- Рекомендуемое расширение: GitLens — для анализа истории и авторства.
